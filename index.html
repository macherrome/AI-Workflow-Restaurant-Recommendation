Section 04

Data Cleaning & AI Enrichment

หลังจาก scrape raw data มาจาก Google Places แล้ว Claude AI ทำหน้าที่ clean, normalise และ enrich ข้อมูลก่อนนำไป score

🧹
1. Merge & Deduplicate
ลบรายการซ้ำด้วย placeId · รวม field ที่ขาดหายจาก raw data
🏷️
2. Normalise Fields
ปรับ priceLevel → ราคา/คน · จัดหมวด foodType · แปลง hours เป็น readable
🤖
3. Claude Enrichment
Claude วิเคราะห์ 6 fields ต่อร้าน: pros · cons · risk · uniqueness · noiseLevel · serviceSpeed
📊
4. Store to Sheets
บันทึก Clean data แยก tab ต่อย่าน (5 tabs) ใน Google Sheets พร้อม upsert ด้วย placeId
Claude Enrichment Prompt claude-sonnet-4-6

  "pros": "...",          // ข้อดีหลัก 1 ประโยค
  "cons": "...",          // ข้อเสียหลัก 1 ประโยค  
  "risk": "ต่ำ|กลาง|สูง", // ความเสี่ยงในการพา 8-12 คนไป
  "uniqueness": "...",    // จุดเด่นที่แตกต่างจากร้านทั่วไป
  "noiseLevel": "เงียบ|ปานกลาง|เสียงดัง",
  "serviceSpeed": "เร็ว|ปานกลาง|ช้า"
}
📋 Clean Data Fields (19 columns): name · area · foodType · rating · reviews · price · address · placeId · travelNote · hours · groupOk · sourceUrl · notes · scoreTotal · pros · cons · risk · noiseLevel · serviceSpeed (🤖 enriched by Claude)
Section 05

Data Sources

แหล่งข้อมูลที่ใช้ในระบบ — ตรวจสอบย้อนกลับได้จาก 2 แหล่งหลัก

📍
Source 1 — Google Places API (via Apify)
ข้อมูลร้านอาหาร: ชื่อ, ที่อยู่, พิกัด, เวลาเปิด, priceLevel, placeId · scrape 20 ร้านต่อย่าน ใน radius 900m จาก BTS
🔗 apify.com/compass/crawler-google-places
⭐
Source 2 — Google Maps Reviews
ข้อมูลรีวิว: totalScore (Google rating), reviewsCount · เก็บพร้อมกันผ่าน Apify actor เดียวกัน — place data และ review data แยกกัน
🔗 maps.google.com
📊
Data Storage — Google Sheets
เก็บ Raw data (20 fields) และ Clean data (19 fields) แยก tab ต่อย่าน รวม 10 tabs + Scoring tab · upsert ด้วย placeId
🔗 เปิด Google Sheets Dataset
